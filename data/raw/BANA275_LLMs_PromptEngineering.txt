What is a Language Model?
A probability distribution over sequences of words

P( next word | previous words )
"The cat sat on the ___" → mat (0.4), floor (0.2)...
Given context, predict what comes next
Trained on massive text data (internet, books, code)
Learn patterns, grammar, facts, reasoning implicitly
"Large" = billions of parameters learning these patterns

Pre-training + Fine-tuning Paradigm

Phase 1: Pre-training
• Train on massive unlabeled text
• Learn language patterns, facts
• Self-supervised: predict next word
• Expensive: millions of $$$

Phase 2: Fine-tuning
• Adapt to specific tasks
• Smaller labeled datasets
• RLHF: align with human preferences
• Much cheaper than pre-training
This is why you can use GPT-4 without training from scratch

The Transformer Architecture

Encoder-Only
BERT

Decoder-Only

Encoder-Decoder
T5, BART
Most modern LLMs (GPT, Claude, Gemini) are decoder-only
Key Components:
• Self-Attention: Each word looks at all other words
• Feed-Forward Networks: Process attended information
• Layer Stacking: Repeat many times (GPT-4 has 100+ layers)

How do LLMs work?
Autoregressive Generation
LLMs generate text one token at a time

"The quick brown" → fox
"The quick brown fox" → jumps
"The quick brown fox jumps" → over
"The quick brown fox jumps over" → the
Key Insight:
Each new token is added to context for next prediction
Model can only see past tokens — never the future
This is why LLMs can produce coherent long-form text

Self-Attention: The Key Intuition

"The cat sat on the mat because it was tired."
When processing "it" → attention focuses on "cat" (not "mat")
The Spotlight Analogy
For each word, a spotlight highlights relevant context
Brightness = attention weight (how much to focus)
Happens in parallel — that's why transformers are fast

Self-Attention: Another Example
Server, can I have the check?”

“Looks like I just crashed the server.”

The word server here means two very different things, which we humans can easily disambiguate by looking at surrounding words. Self-attention allows a neural network to understand a word in the context of the words around it.

Self-attention help neural networks disambiguate words, do part-of-speech tagging, entity resolution, learn semantic roles and a lot more.

Prompt Engineering Principles:

Conciseness — Short, focused instructions
Guidance — Provide examples of desired output
Consistency — Control style, tone, detail level

System vs User Prompts

System Prompt
• Sets overall behavior
• Defines persona/constraints
• Persists across conversation
• Hidden from end users

User Prompt
• The actual request
• Changes each turn
• Visible in conversation
• What the user types

Example:
System: "You are a helpful coding assistant."
User: "Write a function to reverse a string"

Zero-Shot Prompting
Asking the model to perform a task without providing examples

Definition
Asking the model to perform a task without providing examples.
The model relies entirely on its pre-trained knowledge.
Example:
"Translate the following sentence into French: 'Knowledge is power.'"
Best for: Simple, well-understood tasks where the model has strong prior knowledge
9
Few-Shot Prompting
Providing examples within the prompt to guide the model

Example:
Translate the following sentences into Spanish:
- "Good morning." → "Buenos días."
- "How are you?" → "¿Cómo estás?"
- "Thank you very much." → "Muchas gracias."

Now translate: "See you tomorrow."
Best for: Complex tasks requiring specific format or style guidance

Chain-of-Thought Prompting
Encouraging the model to think through problems step-by-step

Purpose: Improves reasoning and complex problem-solving
Example:
"Calculate the total cost of 15 items at $7 each, with 5% tax.
Show your work step-by-step."
Applications:
Math & Logic problems | Story understanding | Fact-checking | Complex query answering
13
Chain-of-Thought: Example
"How many legs do four cats and two birds have?"

Without CoT
Model directly answers:

"20 legs."

(May be wrong!)

With CoT
"Let me work through this:
• 4 cats × 4 legs = 16
• 2 birds × 2 legs = 4
• Total: 16 + 4 = 20 ✓"
The magic phrase: "Let's think step by step"
14
Tree-of-Thought Prompting
Exploring multiple reasoning paths before arriving at a conclusion

CoT: Linear
Step 1 → Step 2 → Step 3
         ↓
     Answer

ToT: Branching
     Problem
    /   |   \
  A     B     C
 / \   / \
A1 A2 B1 B2
        ↓
    Best Path
Best for: Decision-making, brainstorming, optimization problems
15
Tree-of-Thought: Example
"How can we arrange a dinner party so everyone has a good time?"

Guest List

Close friends only
Mix of friends & acquaintances

Menu

Full-course dinner
Potluck style

Seating

Assigned seats
Open seating

Entertainment

Organize games
Hire musician
Open discussion
Each branch is evaluated → Best combination selected → Optimal plan created
16
Tree-of-Thought: Example 2

Prompt:
"You have a 5-liter jug and a 3-liter jug. Measure exactly 4 liters.
Use Tree-of-Thought: consider all possible actions at each step."

Response:
Branch 1: Fill 5L jug first          Branch 2: Fill 3L jug first
    ↓                                     ↓
Pour into 3L → 2L left in 5L         Pour into 5L → repeat...
    ↓                                     ↓
Empty 3L, pour 2L into 3L            (longer path)
    ↓
Fill 5L again
    ↓
Pour into 3L (has 2L space)
    ↓
5L jug now has exactly 4L ✓

Branch 1 is optimal: 6 steps vs 8+ steps

Self-Consistency
Sample multiple reasoning paths, take majority vote

Same prompt → Run 5 times → Different paths

Path 1: 42 ✓    Path 2: 42 ✓    Path 3: 38    Path 4: 42 ✓    Path 5: 40

→ Final Answer: 42 (3/5 majority)
Trade-off: Higher accuracy but 5x the cost and latency
Best for: High-stakes decisions where accuracy matters most

ReAct: Reasoning + Acting
Interleave thinking with tool use

Thought: I need to find Apple's stock price
Action:  search("AAPL stock price")
Observation: AAPL trading at $178.50
Thought: Now I can calculate market cap...
Action:  calculate(178.50 * 15.5B)
Thought: Model reasons what to do | Action: Call a tool | Observation: Result feeds back
26
ReAct: Example 1

Question: "What is the GDP per capita of the country where the Eiffel Tower is located?"

ReAct Process:
Thought 1: I need to find which country the Eiffel Tower is in.
Action 1:  search("Eiffel Tower location")
Observe 1: The Eiffel Tower is located in Paris, France.

Thought 2: Now I need to find France's GDP per capita.
Action 2:  search("France GDP per capita 2024")
Observe 2: France's GDP per capita is approximately $44,408.

Thought 3: I have the answer.
Action 3:  finish("The GDP per capita of France, where the
           Eiffel Tower is located, is ~$44,408.")
21
ReAct: Example 2

Task: "Find the best-rated Italian restaurant near me and make a reservation for 2 at 7pm."

ReAct Process:
Thought 1: I need to search for Italian restaurants nearby.
Action 1:  search_restaurants(cuisine="Italian", sort="rating")
Observe 1: Top result: "Trattoria Milano" - 4.8★, 0.5 miles

Thought 2: This looks good. Check if they have availability.
Action 2:  check_availability("Trattoria Milano", time="7pm", party=2)
Observe 2: Available: 6:45pm, 7:15pm, 8:00pm

Thought 3: 7:15pm is closest. Make the reservation.
Action 3:  make_reservation("Trattoria Milano", "7:15pm", party=2)
Observe 3: Confirmed! Reservation #12345

Why LLMs Are Vulnerable
No true understanding: LLMs predict tokens, not "understand" intent
Trained on everything: Including adversarial content
Helpful by design: Will try to help even malicious requests
Context is text: System prompts can be overridden

Key Insight: Security means appropriately constraining helpfulness
32
Common Attack Patterns

Prompt Injection
"Ignore previous instructions..."

Role-Playing
"Pretend you're an evil AI..."

Context Manipulation
Hidden instructions in user data

Encoding Tricks
Base64, pig latin, other languages

Gradual Escalation
Start innocently, push boundaries
33
Defense Strategies: Guardrails

Input
Validate, sanitize, filter attacks

Prompt
Clear boundaries, instruction hierarchy

Model
RLHF training, refusal patterns

Output
Content filtering, PII detection

Reality: No perfect defense. Security is a cat-and-mouse game.
Defense in depth + monitoring + rapid response
34
Best Practices for Secure Prompts
Separate data from instructions
Use delimiters: <user_input>...</user_input>
Instruct model to be skeptical of user content
Validate outputs before programmatic use
Log and monitor for unusual patterns
Test adversarially before deployment
Assume users will try to break your system

Next Module: RAG
Retrieval-Augmented Generation
The problem with LLMs alone:
• Knowledge cutoff — can't know recent events
• Hallucinations — makes up facts confidently
• No access to private/proprietary data
RAG solves this by:
• Retrieving relevant documents from a knowledge base
• Injecting them into the prompt as context
• Generating answers grounded in real sources
